Privacy Policy — Dual Preview

Last updated: July 2026

Dual Preview is a Chrome extension that displays websites in mobile and desktop view side by side. This policy describes what data the extension accesses and how it is used.

The short version

Websites you preview load directly in your browser — your browsing traffic never passes through our servers. The extension does not collect, store, or transmit the content of pages you visit or the URLs you browse. The only data that leaves your device is anonymous feature-usage analytics, and — only if you actively use the AI features — the content of your AI questions.

Data stored locally

The following is saved on your device only (in chrome.storage.local and the extension page's local storage) and is never sent anywhere:

  • Panel layout, open tabs, and their navigation history inside the extension
  • Selected background color or image, including custom images you upload
  • Scaling mode, panel sizes, captions, pinned tools, and keyboard-shortcut preferences
  • Audio mute preference

Permissions used

  • Access to websites (host permissions) — required to load any website you choose into the preview panels and to run the device-emulation scripts inside them. The extension does not read or collect content from pages you are not previewing.
  • declarativeNetRequest / declarativeNetRequestWithHostAccess — modifies the User-Agent request header so previewed sites serve their mobile or desktop version, and removes embedding restrictions only for pages loaded inside preview panels. Rules are scoped to previewed domains and cleaned up when the extension's tab is closed.
  • history — provides URL autocomplete suggestions in the address bars. History is read locally and never stored or transmitted.
  • debugger — used only when you enable the Network Speed (2G/3G/4G) simulation for a panel; Chrome shows its own notification bar while this is active. Detached automatically when throttling is turned off.
  • browsingData — used only when you click "Clear cache" for a panel; clears cached data for that site's origin.
  • webRequest — powers the Analytics Detector (BETA): it observes, locally in your browser, analytics requests made by pages inside your preview panels so it can display them to you. Nothing observed is stored or transmitted.
  • cookies — removes deployment-pinning cookies for the extension's own new-tab page so it always loads the latest version.
  • storage / unlimitedStorage — stores the local preferences listed above, including custom background images.

AI features (optional)

The AI Inspect and Inspector chat features send your question together with the technical context of the element you selected (its HTML snippet and CSS styles) to an AI model provider. This happens only when you actively use these features. Requests are routed through the extension's own proxy endpoint to OpenAI; the proxy does not log or store your conversations, and no account or API key is required from you.

Analytics

The extension uses PostHog (EU cloud servers) to collect anonymous usage analytics under a randomly generated installation ID — e.g. which features are used and how often the extension is opened. Automatic event capture and session recording are disabled. No personally identifiable information is collected, and no URLs or content of websites you preview are tracked. You can opt out at any time by contacting us.

Third-party services

  • Favicon service — address-bar suggestions display site icons fetched from Google's public favicon service; only the domain of the suggested site is sent for this purpose.
  • New-tab page and configuration — the extension's built-in new-tab page and a small configuration file are hosted on Vercel and loaded like a regular web page.

Data sharing

Dual Preview does not sell or transfer any user data to third parties. Anonymous analytics data is processed by PostHog solely for product improvement purposes.
